> 作成日: 2026-04-23  更新: 2026-04-23  
> 対象バージョン: 次期リリース  
> 関連 ISSUE: #UI-01 テーマ対応 / #UI-02 設定ページ再構成 / #UI-03 ダッシュボードリデザイン / #UI-04 ヘルプページ

---

## 背景・目的

転送ロジックの安定化が完了したフェーズで、エンドユーザーが使いやすい UI へ改善する。  
主要課題は以下の 3 点：

1. システムテーマ非追随（ダークモードが使えない）
2. 設定ページの「上級者向け設定」に一般ユーザー向け設定（グラフ表示等）が混在
3. ダッシュボードが一般ユーザーには情報過多で、重要操作（転送開始）が下部にある

## 1. テーマ対応（ISSUE-UI-01）

### 現状
- `DashboardApp.razor` に `PaletteLight` のみ定義
- ダークモード・システム追随なし

### 設計方針
- テーマモードを 3 択にする: **ライト / ダーク / システムに従う**（デフォルト: システムに従う）
- 選択値は設定に永続化（`config.json` の `ui.themeMode`）

### ダークテーマ カラーパレット
ARBORデザイン（`arbor-spec-design-b_2.html`）の配色を MudBlazor `PaletteDark` にマッピングする。

| 役割 | 値 | 参照元変数 |
|------|-----|-----------|
| Background (最暗) | `#0c0e14` | `--bg0` |
| Background (メイン) | `#141720` | `--bg2` |
| Surface (カード等) | `#1a1e2e` | `--bg3` |
| AppBar Background | `#0f1117` | `--bg1` |
| Primary | `#818cf8` | `--indigo` |
| Primary Darken | `#6366f1` | `--indigo-d` |
| Primary Lighten | `#a5b4fc` | `--indigo-l` |
| Secondary | `#94a3b8` | `--text2` |
| Success | `#4ade80` | `--green` |
| Warning | `#fbbf24` | `--amber` |
| Error | `#f87171` | `--red` |
| Info | `#818cf8` | `--indigo` |
| Text Primary | `#e2e8f0` | `--text1` |
| Text Secondary | `#94a3b8` | `--text2` |
| Divider | `#252c42` | `--border2` |
| Table Lines | `#1e2336` | `--border` |
| Drawer Background | `#0a0c12` | サイドバー色 |

### 実装対象ファイル
- `DashboardApp.razor` — `PaletteDark` 追加、`_isDarkMode` / `_themeMode` フィールド追加、テーマ切替 UI
- `SettingsPage.razor` — 表示設定セクションにテーマ切替 `MudSelect` 追加
- `IConfigurationService` / `ConfigurationService` — `UiThemeMode` の読み書き追加
- `config.json` スキーマ — `ui.themeMode` キー追加

### MudBlazor 実装メモ
```razor
<MudThemeProvider Theme="_theme" @bind-IsDarkMode="_isDarkMode" />
```
- `DefaultScrollbar="true"` も指定してスクロールバーもテーマ追随させる
- システム追随は `MudThemeProvider` の `ObserveSystemThemeChange` を利用（または JS interop で `prefers-color-scheme` 検出）

## 2. 設定ページ再構成（ISSUE-UI-02）

### 現状の問題
- 「⚙️ 上級者向け設定（並列数・チャンク・タイムアウト）」という 1 つの `MudExpansionPanel` に以下が混在:
  - グラフ表示設定（一般ユーザー向け）
  - 転送パラメータ（中級者向け）
  - レートベース制御エンジン切替（上級者向け）
  - レートパラメーター詳細・HybridRateController（上級者専用・実験的）
  - DBリビルドボタン（適切な警告レベルと文言が必要）

### 新セクション構成

```
設定ページ
├── [A] 転送ルート情報（読み取り専用・拡充）
├── [B] 表示設定（常時展開・折りたたみなし）
├── [C] 基本設定（MudExpansionPanel・デフォルト閉）
├── [D] 詳細設定（MudExpansionPanel・デフォルト閉）
└── [E] データ管理（常時展開・折りたたみなし）← 一般ユーザーがアクセスできる位置
```

> **DB初期化は一般設定**  
> 転送先・転送元を変更した場合や「最初からやり直したい」ニーズに応えるため、折りたたみには入れない。  
> ただし誤操作防止のため確認ダイアログ必須。警告レベルは Warning（赤にしない）。

#### [A] 転送ルート情報
現在表示項目に加え、以下をツールチップ（`MudTooltip`）で追加表示：

| フィールド | 表示方法 |
|-----------|---------|
| OneDrive ユーザー UPN | 本文に表示（現状） |
| OneDrive 表示名 | ツールチップ |
| SharePoint サイト名 | 本文に表示 |
| SharePoint サイト URL | ツールチップ（リンク） |
| ドキュメントライブラリ名 | 本文に表示 |
| 転送先フォルダパス | 本文に表示（現状） |
| 転送元フォルダパス | 本文に表示（現状） |

**実装に必要なデータ変更:**  
`DiscoveryConfigDto` / `DiscoveryConfigUpdateDto` に以下フィールドを追加:
- `OneDriveDisplayName` (OneDriveDiscoveryResult.DisplayName から保存)
- `SharePointSiteDisplayName`（ウィザードの `_selectedSite.DisplayName`）
- `SharePointSiteWebUrl`（ウィザードの `_selectedSite.WebUrl`）
- `SharePointDriveDisplayName`（ウィザードの `_selectedDrive.DisplayName`）

保存タイミング: `DriveDiscoveryPage.razor` / `SharePointDiscoveryPage.razor` の SaveAndContinueAsync

#### [B] 表示設定（折りたたみなし、一般ユーザー向け）

| 設定項目 | コントロール |
|---------|------------|
| テーマ | `MudSelect`（ライト / ダーク / システムに従う） |
| グラフ列数 | `MudSelect`（1〜4列） |

> **注意**: メトリクスグラフの表示 ON/OFF はダッシュボードの「詳細情報」タブ内の各グラフカードヘッダーに `MudSwitch` を配置し、カード単位で操作する。設定ページへのグローバルスイッチは廃止。

#### [C] 基本設定（折りたたみ、推奨度高い順）

| 設定項目 | 推奨度の根拠 |
|---------|------------|
| 最大並行転送数 | 最も一般的に調整する |
| タイムアウト (秒) | ネットワーク環境により要調整 |
| リトライ回数 | デフォルト(3)でほぼ十分 |

#### [D] 詳細設定（折りたたみ、上級者向け）

順序（推奨度・影響度 高→低）:

1. 転送制御エンジン（レートベース制御 ON/OFF）
2. レートベース制御パラメーター（エンジン ON 時）
   - 最大並行数 / レート上限（安全キャップ）
   - インフライト閾値
   - 短期ウィンドウ (秒)
   - 中期ウィンドウ (秒)
   - 緊急減速閾値 (429 率 %)
   - 緩減速閾値 (429 率 %)
   - 減衰係数 K / 最小・最大減衰率
3. HybridRateController（`MudExpansionPanel` でさらに折りたたみ、「実験的」バッジ付き）
4. チャンクサイズ (MB)
5. 大ファイル閾値 (MB)
6. 最大並行フォルダ作成数

#### [E] データ管理

| 変更前 | 変更後 |
|-------|-------|
| ボタンラベル「DBリビルド」 | 「データベースを初期化」 |
| `Color.Error` の警告 | `Color.Warning` / `Severity.Warning` |
| 警告文（現状不明だが強い） | 「転送済みの記録がすべて削除され、次回実行時にすべてのファイルが再転送対象になります。完了済みファイルも含めて再転送したい場合にのみ実行してください。」 |
| 確認ダイアログ（現状不明） | `MudMessageBox` で「初期化する」「キャンセル」の 2 択 |

## 3. ダッシュボードリデザイン（ISSUE-UI-03）

### 現状の問題
- 転送開始ボタンがページ下部のカード内にあり、操作しにくい
- 一般ユーザー向けには情報過多（レートモニター、429 率、インフライト数etc.）
- 「総経過（壁時計）」という表現がわかりにくい
- グラフが概要と混在

### 新レイアウト：タブ構造

```
ダッシュボードページ
├── ヘッダー行（フェーズラベル・ルートチップ・転送開始/停止ボタン）← 常時表示
├── 進捗バー ← 常時表示
└── [Tab1: 概要] / [Tab2: 詳細情報] / [Tab3: ログ]
```

#### ヘッダー行（タブの外・常時表示）

```
[転送ダッシュボード] [フェーズチップ] [ルートチップ]    [▶ 転送開始] [■ 停止]
```

- 転送開始/停止ボタンをヘッダー右端に移動（現在のアイコンボタン位置に統合）
- フェーズラベルを人間語に変換（後述）

#### フェーズラベル 人間語化マッピング

| 内部値 | 表示ラベル | チップ色 |
|-------|-----------|---------|
| `crawling` | ファイルを確認中... | Info |
| `folder_creation` | フォルダを作成中... | Warning |
| `transferring` | 安定転送中 | Success |
| `transferring` + 短期429率 > 緊急閾値 | 流量制御中 | Error |
| `transferring` + 短期429率 > 緩閾値 | 速度調整中 | Warning |
| `completed` | 移行完了 ✓ | Success |
| `stopped` | 停止済み | Default |
| `(空)` | 待機中 | Default |

#### [Tab1: 概要]（一般ユーザー向け）

1. **ステータスカード行**（完了 / 処理中 / 待機 / 失敗 / 永久失敗 / 合計）
2. **経過時間行**（実稼働時間 + 総経過時間 + 残り予測 ETA）  
   ※「壁時計」→「総経過時間」に変更
3. **一行ログ**（「動いてる感」演出）
   - 最新の転送中ファイル名のみ表示（パス省略、ファイル名のみ）
   - 429 / エラー系は表示しない
   - フェード in/out アニメーション（CSSアニメーション）
   - 転送中以外（クロール中・フォルダ作成中）は対応するメッセージを表示
4. **移行完了サマリー**（`phase == completed` 時のみ）

#### [Tab2: 詳細情報]（上級者向け）

1. **レートモニター**（`_useRateControl` 時のみ）
   - 制御状態チップ / レート上限(req/s) / インフライト数 / 短期429率 / 中期429率
2. **実稼働統計**
   - 現在並列数 / リトライ総数 / エラー率 / 転送済みデータ
3. **メトリクスグラフ**（各グラフカードで個別に表示/非表示を切り替え）
   - スループット(files/min) / スループット(bytes/sec) / レートリミット率 / 並列実行数
   - 表示状態は設定ページのグローバルスイッチではなく、カード単位の UI 状態として管理する
4. **最近の失敗**（最大5件）

#### [Tab3: ログ]

- 現行の `LogsPage` の内容をタブ内に表示（または LogsPage へのリンク）
- 実装コスト次第でフェーズ2として後回し可

### 一行ログの実装メモ

**データソース候補:**  
`ITransferStateDb` の `GetRecentProcessingAsync()` 的なメソッド（要追加）、または既存の `TransferJob` テーブルから `Status = Processing` のもの最新1件を 1秒ポーリングで取得。

**表示ロジック:**  
```csharp
// Path + Name からファイル名のみ抽出
var fileName = Path.GetFileName(item.Name ?? item.Path);
```

**アニメーション:**  
CSS `transition: opacity 0.3s ease-in-out` でフェード切り替え。Blazor 側は `_logVisible` フラグを flip することでトリガー。

## 4. ヘルプページ（ISSUE-UI-04）

### 概要

ドロワーに「ヘルプ」リンクを追加し、セットアップ手順・操作方法・設定リファレンス・バージョン情報・更新導線を 1 ページにまとめる。外部ブラウザを開かずアプリ内で完結させる。

### ドロワーへの追加位置

```
Drawer
├── ダッシュボード
├── 設定
├── 接続テスト
├── ログ
├── ──────
├── ヘルプ          ← 追加（? アイコン）
└── セットアップをやり直す
```

### ページ構成（MudTabs または MudExpansionPanels）

```
ヘルプページ
├── [A] セットアップ
├── [B] 基本的な操作方法
├── [C] 設定リファレンス（上級者向け）
└── [D] バージョン情報 / 更新確認
```

---

#### [A] セットアップ

| 手順 | 内容 |
|------|------|
| 1. Azure App 登録 | Azure AD でアプリ登録 → Client ID / Secret / Tenant ID を取得 |
| 2. ウィザード起動 | アプリ初回起動時に自動表示。「セットアップをやり直す」で再実行可能 |
| 3. 認証情報入力 | Client ID / Client Secret / Tenant ID を入力 |
| 4. OneDrive 選択 | 移行元ユーザーの OneDrive ドライブを選択 |
| 5. SharePoint 選択 | 移行先サイト・ライブラリ・フォルダを選択 |
| 6. 転送開始 | ダッシュボードの「転送開始」ボタンを押す |

表示形式: 番号付きステッパー表示（`MudStepper` または ol リスト）

---

#### [B] 基本的な操作方法

**ダッシュボード**

| 操作 | 方法 |
|------|------|
| 転送を開始する | ヘッダー右端の「▶ 転送開始」ボタンを押す |
| 転送を停止する | ヘッダー右端の「■ 停止」ボタンを押す |
| 進捗を確認する | 「概要」タブのステータスカードと進捗バーを見る |
| 失敗ファイルを確認する | 「詳細情報」タブの「最近の失敗」セクションを参照 |
| 詳細ログを見る | 「ログ」タブ、またはドロワーの「ログ」ページ |

**設定**

| 操作 | 方法 |
|------|------|
| 転送先・転送元を確認する | 設定の「転送ルート情報」セクションを参照 |
| 設定を変更した後に適用する | ページ下部の「保存」ボタンを押す |
| 最初から移行をやり直す | 設定の「データ管理」→「データベースを初期化」 |
| セットアップ情報を更新する | ドロワーの「セットアップをやり直す」 |

#### [C] 設定リファレンス（上級者向け）

##### 基本設定

| 項目 | 説明 | デフォルト | 推奨範囲 |
|------|------|-----------|---------|
| 最大並行転送数 | 同時に実行するファイル転送の上限数。大きくするほど速いが、Graph API の呼び出しレートも上がるため 429 エラーが増えやすい。ネットワーク品質に応じて調整する。 | 16 | 4〜32 |
| タイムアウト (秒) | 1 リクエストあたりの上限時間。大容量ファイルのアップロード中に到達するとリトライになる。大ファイルが多い場合は大きめに設定する。 | 300 | 120〜600 |
| リトライ回数 | 一時的なエラー（5xx / ネットワーク断）が発生した際の再試行回数。最大回数を超えた場合は「失敗」ステータスになる。 | 3 | 2〜5 |

##### 詳細設定（レートベース制御エンジン）

レートベース制御エンジンは、Graph API からの応答（特に HTTP 429 レスポンス）を監視しながら転送レートを自動調整します。従来の「並列数固定」方式に比べ、API 側の状況に適応できます。

| 項目 | 説明 | デフォルト |
|------|------|-----------|
| レートベース制御エンジン | ON: req/s 主体の新エンジンで API 状況に適応。OFF: 旧来の並列数固定エンジン（シンプル・予測可能）。 | ON |
| 最大並行数 / レート上限（安全キャップ） | レートベース制御がどれだけ加速しても超えない上限。ネットワーク帯域上限の目安として設定する。 | 16 |
| インフライト閾値 | 同時処理リクエスト数がこの値を超えると新規リクエストの発行を抑制する。並列数の上限バッファ。 | 32 |
| 短期ウィンドウ (秒) | 直近 N 秒間の 429 率を監視するウィンドウ幅。小さいほど即座に反応するが不安定になりやすい。 | 5 |
| 中期ウィンドウ (秒) | 中〜長期の 429 率を監視するウィンドウ幅。過去の傾向を加味した安定した判断に使う。 | 30 |
| 緊急減速閾値 (429 率 %) | 短期 429 率がこの値を超えると即座に大幅減速する。急激なレートリミットに対応するための緊急ブレーキ。 | 10 |
| 緩減速閾値 (429 率 %) | 短期 429 率がこの値を超えると緩やかに減速する。通常の流量調整に使う。 | 3 |

##### 詳細設定（ファイル転送）

| 項目 | 説明 | デフォルト |
|------|------|-----------|
| チャンクサイズ (MB) | 大容量ファイルを分割アップロードする際の 1 チャンクのサイズ。Graph API の LargeFileUpload に使用。大きすぎると 1 チャンクのタイムアウトリスクが増す。 | 10 |
| 大ファイル閾値 (MB) | これより大きいファイルをチャンクアップロードに切り替える閾値。Graph API の制限（4 MB）以上を推奨。 | 4 |
| 最大並行フォルダ作成数 | フォルダ構造の作成フェーズで同時に作成するフォルダ数の上限。 | 4 |

#### [D] バージョン情報 / 更新確認

**表示内容:**

- 現在のバージョン（アセンブリバージョン）
- ビルド日時（オプション、`AssemblyInformationalVersion` に埋め込み）
- GitHub Releases ページへのリンク
- **新バージョンが利用可能な場合のみ**バナー表示

**更新チェックの仕組み:**

```
起動時 or「更新を確認」ボタン押下
    ↓
GitHub API: GET https://api.github.com/repos/scottlz0310/cloud-migrator/releases/latest
    ↓
{tag_name: "v0.8.0"} を取得
    ↓
現在バージョン (v0.7.0) と比較
    ↓
新バージョンあり → リリースノートと GitHub Releases へのリンクを表示
新バージョンなし → 「最新バージョンを使用中」と表示
```

**実装メモ:**

- `HttpClient` は `IHttpClientFactory` 経由で注入（User-Agent ヘッダー必須: GitHub API 要件）
- ネットワーク不通・API エラー時はサイレントに失敗（更新チェックは補助機能）
- バージョン比較は `Version.Parse` で行う（プレフィックス `v` を除去してから比較）
- リリース URL: `https://github.com/scottlz0310/cloud-migrator/releases/tag/{tag_name}`

**UI 状態:**

| 状態 | 表示 |
|------|------|
